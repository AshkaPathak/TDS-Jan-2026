## Problem Summary
The task required using GitHub Copilot CLI to answer a question inside an `asciinema` recording. The recording had to include a required session marker and the Copilot prompt, and the full `session.cast` JSON had to be submitted to the portal.

## Required Command Sequence
The following commands had to be executed in this exact order inside the recording:

```bash
echo 'COPILOT_A9NB7Z66'
gh copilot -p "How many sides does a hexagon have?"
```

## Issues Identified
1. `gh` (GitHub CLI) was not installed initially, causing `zsh: command not found: gh`.
2. The `gh copilot ask` syntax was invalid for this Copilot CLI version, requiring the `-p/--prompt` option.
3. `session.cast` already existed from Q12, so a new output file name was required to avoid overwriting.

## Strategy to Fix
- Install GitHub CLI (`gh`) and authenticate.
- Use non-interactive Copilot CLI mode with `-p` (prompt).
- Record to a new cast file (`session_q13.cast`) to avoid conflicts with earlier recordings.
- Submit the complete cast JSON content by copying the output of `cat session_q13.cast`.

## Implementation Steps

### 1. Install and Authenticate GitHub CLI
